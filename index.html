The core Distinction

Data Type vs Symbol Type

Data Type (Content)

What values can it hold? (int, float). Determines memory size.

Symbol Type (Role)

How is it used? (VARIABLE, FUNCTION). Determines parse validity.

Handling Left Recursion

Infinite loops are avoided through **Precedence Climbing** and **Grammar Refactoring**. For arithmetic expressions, we use iterative loops instead of pure recursion to ensure $LL(k)$ stability.

Trap Questions

"Why a flat Symbol Table?"

Strategy: Frame it as a performance/boundary choice. "Our focus was Frontend Syntax. A flat $O(1)$ lookup ensures clean JTree visualization within the project timeline, delegating nested scope management to the Semantic/Optimization phase."

"Why no type-mismatch errors?"

Strategy: Use the "Syntax vs Semantics" boundary. "Our parser validates structure. void x = 10; is syntactically perfect even if semantically flawed. We focused on building a robust Syntax Analyzer first."
